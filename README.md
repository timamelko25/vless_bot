# VLESS VPN Telegram Bot

## Общая идея

**VLESS VPN Telegram Bot** — это приложение на базе Telegram, предназначенное для предоставления пользователям безопасных и надежных VPN-сервисов на протоколе VLESS. Бот позволяет пользователям приобретать, управлять и активировать VPN-ключи, просматривать баланс и участвовать в реферальной программе для получения бонусов. Администраторы имеют доступ к полноценной панели управления для работы с пользователями, серверами, ключами, промокодами и отправки массовых сообщений. Проект разработан с акцентом на масштабируемость, безопасность и удобство использования, с применением современных технологий и контейнеризированной архитектуры.

### Основные функции
- **Для пользователей**:
  - Покупка VPN-ключей (150 ₽ за ключ, поддержка до 3 устройств с лимитом трафика 100 ГБ).
  - Просмотр и управление личными VPN-ключами.
  - Активация промокодов для пополнения баланса.
  - Участие в реферальной программе (20% бонус от платежей приглашенных пользователей).
  - Доступ к инструкциям по настройке для iOS, Android, macOS и Windows.
- **Для администраторов**:
  - Управление серверами (добавление/удаление).
  - Генерация и управление VPN-ключами.
  - Создание и распространение промокодов.
  - Отправка массовых сообщений пользователям или администраторам.
  - Обновление баланса пользователей и управление сроками действия ключей.
- **Автоматизация подписки**:
  - Ежедневная проверка сроков действия ключей с напоминаниями за 3 дня до списания.
  - Автоматическое продление подписки для пользователей с достаточным балансом.
  - Деактивация ключей при недостаточном балансе.

## Технологический стек

Проект построен на современном и масштабируемом технологическом стеке, оптимизированном для асинхронных операций и контейнеризированного развертывания:

- **Язык программирования**: Python 3.12
- **Фреймворк**: FastAPI (для обработки вебхуков и API-эндпоинтов)
- **Telegram-бот**: Aiogram 3.x (асинхронный фреймворк для Telegram Bot API)
- **Очередь сообщений**: RabbitMQ (для асинхронной рассылки сообщений)
- **База данных**:
  - PostgreSQL (для хранения данных пользователей, ключей, платежей и промокодов)
  - SQLAlchemy с AsyncPG (для ORM и асинхронных операций с базой данных)
- **Кэш/Хранилище**: Redis (для хранения состояний FSM Aiogram)
- **Планировщик**: APScheduler (для управления продлением подписок и напоминаний)
- **Контейнеризация**: Docker и Docker Compose
- **Логирование**: Loguru (для структурированного логирования)
- **CI/CD**: GitHub Actions (для автоматизированного развертывания)

## Структура проекта

Проект организован в модульной структуре, основные файлы и директории:

- `app/` — Основной код приложения:
  - `config.py` — Загрузка конфигурации из `.env` и инициализация бота, планировщика и брокера сообщений.
  - `database.py` — Настройка SQLAlchemy для асинхронных операций с базой данных.
  - `main.py` — Точка входа для FastAPI-приложения и вебхука бота.
  - `entities/` — Бизнес-логика для пользователей, серверов, ключей, платежей и промокодов.
  - `broker/` — Обработка сообщений RabbitMQ для массовой рассылки.
  - `scheduler/` — Управление подписками и проверка сроков действия ключей.
  - `utils/` — Вспомогательные функции (настройка логирования, удаление сообщений).
  - `external_api/` - Работа с внешним API (VLESS панель)
- `Dockerfile` — Определение Docker-образа на базе Python 3.12 для бота.
- `docker-compose.yml` — Оркестрация сервисов (бот, PostgreSQL, Redis, RabbitMQ).
- `.env.example` — Шаблон для переменных окружения.
- `.github/workflows/deploy.yaml` — Рабочий процесс GitHub Actions для развертывания.

## Установка и запуск через Docker

Следуйте этим шагам, чтобы настроить и запустить проект с помощью Docker:

### 1. Клонирование репозитория
```bash
git clone https://github.com/timamelko25/vless_bot.git
cd vless_bot
```

### 2. Настройка переменных окружения
```bash
cp .env.example .env
```

### 3. Сборка и запуск с помощью Docker Compose

```bash
docker-compose up --build -d
```

## Запуск тестов

### 1. Запуск тестовых контейнеров

```bash
docker-compose -f docker-compose.test.yml up -d 
```

### 2. Применение тестового окружения

```bash
export $(grep -v '^#' .env.test | xargs)
```

### 3. Запуск тестов

```bash
pytest tests/ -v
```
